# Task ID: 21
# Title: Split Configuration Files and Remove Hardcoded Settings
# Status: pending
# Dependencies: 4
# Priority: high
# Description: Split the large config.template.yaml into separate basic and advanced configuration files, implement robust configuration loading and merging logic, and eliminate all hardcoded settings throughout the codebase.
# Details:
Create two new configuration files: config.yaml (basic user settings like URLs, passwords, API keys, project definitions) and advanced.yaml (technical settings like rate limits, timeouts, batch sizes, worker pool sizes, retry counts with sensible defaults). Implement ConfigurationManager class that loads and merges both files at startup, with basic config taking precedence over advanced settings during conflicts. Add Pydantic models for validation of both BasicConfig and AdvancedConfig schemas. Support environment variable overrides using ${VAR:-default} syntax for both files. Scan entire codebase to identify hardcoded values (connection timeouts, retry counts, batch sizes, API endpoints, etc.) and move them to advanced.yaml with appropriate defaults. Implement backward compatibility by checking for existing config.template.yaml and migrating settings automatically. Add comprehensive configuration documentation with examples and explanations for each setting. Create configuration validation that checks for required fields, validates data types, and provides helpful error messages. Implement hot-reload capability for non-critical configuration changes. Add configuration export functionality to generate template files for new deployments. Ensure thread-safe configuration access using singleton pattern with proper locking.

# Test Strategy:
Create unit tests for ConfigurationManager class covering file loading, merging logic, environment variable substitution, and validation scenarios. Test backward compatibility by verifying existing config.template.yaml files are properly migrated. Create integration tests that verify all hardcoded values have been successfully moved to configuration files by running the application with different configuration combinations. Test configuration validation with invalid inputs to ensure proper error handling and user-friendly messages. Verify environment variable overrides work correctly for both basic and advanced settings. Test hot-reload functionality by modifying configuration files during runtime and verifying changes are applied. Create performance tests to ensure configuration loading doesn't impact application startup time. Test edge cases like missing files, corrupted YAML, circular dependencies, and conflicting settings between basic and advanced configurations.
