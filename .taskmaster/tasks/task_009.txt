**Data Pruning Strategies Implemented:**

1. **Age-Based Pruning** (`prune_old_data()`):
   - Remove data older than specified number of days
   - Support for filtering by specific node types
   - Configurable cutoff timestamps with proper relationship cleanup
   - Dry-run mode for safe analysis before deletion

2. **Orphaned Node Removal** (`prune_orphaned_nodes()`):
   - Identify and remove nodes with no relationships
   - Prevents graph fragmentation and reduces storage overhead
   - Safe deletion with comprehensive error handling

3. **Duplicate Relationship Cleanup** (`prune_duplicate_relationships()`):
   - Detect and remove duplicate relationships between same nodes
   - Preserves the first relationship while removing duplicates
   - Maintains graph integrity while reducing redundancy

4. **Quality-Based Entity Pruning** (`prune_low_quality_entities()`):
   - Remove entities based on configurable quality criteria
   - Minimum relationship count filtering
   - Minimum name length validation
   - Flexible criteria system for custom quality rules

5. **Comprehensive Pruning** (`comprehensive_data_pruning()`):
   - Orchestrates multiple pruning strategies in optimal order
   - Configurable strategy selection (age, orphans, duplicates, quality)

6. **Intelligent Recommendations** (`get_pruning_recommendations()`):
