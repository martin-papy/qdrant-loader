# Task ID: 19
# Title: Enhance Data Source Connectors to Extract and Store Metadata for Graph Relationships
# Status: pending
# Dependencies: 4
# Priority: medium
# Description: Extend the Git, Confluence, and JIRA connectors to extract and persist metadata such as author information, timestamps, cross-references, project relationships, and contextual data to enrich the knowledge graph, without altering the core connector interfaces.
# Details:
Update each data source connector (Git, Confluence, JIRA) to extract relevant metadata during data ingestion. For Git, capture commit author, commit timestamp, file references, and branch/merge relationships. For Confluence, extract page authors, creation/modification timestamps, parent/child page links, and embedded cross-references. For JIRA, use the REST API to retrieve issue metadata including reporter, assignee, creation/update timestamps, project and epic links, and references to related issues (e.g., via 'relates to', 'blocks', or 'duplicates' fields)[1][5]. Design a metadata extraction layer within each connector that collects this information and serializes it into a structured format (e.g., JSON or dict) alongside the main content payload. Ensure this metadata is stored in a way that is accessible to downstream graph processing (e.g., as part of the document object or a dedicated metadata field). Do not modify the public connector interfaces; instead, use internal extension points or hooks. Update documentation to specify what metadata is extracted per source and how it is structured. Ensure compatibility with the configuration management system for enabling/disabling metadata extraction per source.

# Test Strategy:
1. For each connector, ingest a representative sample of data and verify that the extracted metadata (author, timestamps, cross-references, etc.) is present and correctly populated in the output objects. 2. For JIRA, use the REST API to fetch issues and confirm that metadata such as reporter, assignee, timestamps, and linked issues are captured[1][5]. 3. For Confluence, verify that page hierarchy and cross-references are extracted. 4. For Git, check that commit metadata and file relationships are included. 5. Confirm that the metadata is accessible to the graph processing pipeline and that no changes to the public connector interfaces have occurred. 6. Validate that metadata extraction can be toggled via configuration. 7. Review and test documentation updates for accuracy.
